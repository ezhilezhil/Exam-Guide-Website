1. What is SQL Injection?

Answer: SQL Injection is a cyber attack that exploits vulnerabilities in a web application's database query execution. It allows attackers to manipulate the application's SQL queries by injecting malicious SQL code into input fields.

2. How does SQL Injection work?

Answer: SQL Injection works by injecting malicious SQL code into user input fields. For example, an attacker might input `' OR '1'='1` to bypass authentication or access sensitive data in the database.

3. What are the impacts of SQL Injection?

  • Unauthorized access to sensitive data.
  • Data theft, modification, or deletion.
  • Potential compromise of the entire database.

4. How can SQL Injection attacks be prevented?

  • Use parameterized queries or prepared statements.
  • Validate and sanitize user inputs.
  • Implement proper error handling to avoid exposing database details.

5. What are Keyloggers?

Answer: Keyloggers are malicious software or hardware devices designed to record every keystroke made on a device. They are often used to capture sensitive information such as passwords and credit card numbers.

6. How do Keyloggers work?

Answer: Keyloggers operate in the background, monitoring and recording all keyboard input. They can be:

  • Software-based: Installed as malware on a device.
  • Hardware-based: Attached as physical devices like USBs or keyboard connectors.

7. What are the impacts of Keyloggers?

  • Compromise of sensitive information (e.g., passwords, financial data).
  • Facilitation of identity theft.
  • Potential access to corporate or personal accounts.

8. How can Keylogger attacks be prevented?

  • Use anti-keylogging and updated antivirus software.
  • Avoid entering sensitive information on public or shared devices.
  • Regularly update device security patches.

9. What is Wireless Security?

Answer: Wireless security protects wireless communication from unauthorized access and malicious attacks.

10. What are common threats to wireless security?

  • Eavesdropping, unauthorized access, and denial of service (DoS).

11. How can wireless security be enhanced?

  • Use WPA3 encryption.
  • Enable MAC address filtering.
  • Set strong passwords for Wi-Fi networks.

12. What is Cloud Security?

Answer: Cloud security ensures the protection of data, applications, and infrastructure in cloud environments.

13. What are common threats to cloud security?

  • Data breaches, insecure APIs, and insider threats.

14. How can cloud security be enhanced?

  • Implement data encryption.
  • Use multi-factor authentication.
  • Conduct regular security audits.

15. What is Web Security?

Answer: Web security focuses on safeguarding websites and web applications from cyber threats.

16. What are common web security threats?

  • Cross-Site Scripting (XSS), SQL Injection, and Distributed Denial of Service (DDoS).

17. How can web security be improved?

  • Use HTTPS for secure communication.
  • Implement firewalls and intrusion detection systems.
  • Follow secure coding practices.

18. What is RSA Cryptosystem?

Answer: RSA is a widely-used public-key encryption system that relies on the mathematical difficulty of factoring large numbers.

19. How does RSA work?

  • RSA generates a pair of keys: a public key for encryption and a private key for decryption.

20. What is Diffie Hellman Key Exchange?

Answer: Diffie-Hellman is a method for securely exchanging cryptographic keys over a public channel.

21. How does Diffie-Hellman work?

  • It allows two parties to generate a shared secret key without exchanging the key itself.

22. What is Digital Signature Standard (DSS)?

Answer: DSS is a standard for creating and verifying digital signatures, ensuring authenticity and integrity of messages or documents.

23. What is SHA (Secure Hash Algorithm)?

Answer: SHA is a cryptographic hash function designed to take an input and produce a fixed-size hash value. It's widely used for data integrity and digital signatures.
